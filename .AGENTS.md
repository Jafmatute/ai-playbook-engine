## Monorepo
- pnpm workspace. Packages in `apps/*` and `packages/*`.
- Path aliases in `tsconfig.base.json`:
  `@ai-playbook-engine/{core,infrastructure,notion,ai-providers,config,shared,testing}` ‚Üí `packages/{*}/src`
- Node >=20, pnpm >=9. TypeScript ES2022, ESNext modules, bundler resolution.

## Architecture & boundaries

### `apps/api`
HTTP entry point. Endpoints for sync, playbooks, audits, results, config, webhooks.

### `apps/cli`
CLI interface. Commands: `playbook sync`, `validate`, `audit`, `recommend`, `status`.

### `apps/worker`
Off-request processing: scheduled Notion sync, content processing, embeddings, long audits, automation, retries. **Not implemented yet** ‚Äî reserved boundary.

### `packages/core`
Domain + application + use cases + domain services + ports + contracts.
**Must NOT depend on:** Fastify, Prisma, Notion, OpenAI, Anthropic, filesystem, env vars.

### `packages/infrastructure`
Technical implementations: persistence, repositories, logging, filesystem, queues, cache, system services.

### `packages/notion`
Notion adapter: client, page/database reading, block conversion, sync, rate limits, external‚Üíinternal mapping.

### `packages/ai-providers`
AI provider adapters: OpenAI, Anthropic, Google, local models. Core must not depend on any.

### `packages/config`
Centralized validated config: env vars, DB, Notion, providers, execution settings.

### `packages/shared`
Shared utilities: types, IDs, dates, results, base errors. **Do not let this become a junk drawer.**

### `packages/testing`
Test tooling: builders, factories, fixtures, fakes, controlled mocks, integration helpers.

## Dependency management
- Shared devDependencies across the monorepo (build tools, formatters, linters, etc.) ‚Üí `pnpm add -Dw ...` at root.
- Dependencies specific to a single app/package (express, zod, openai, etc.) ‚Üí install inside that workspace (`apps/api`, `packages/notion`, etc.) with `pnpm add` (no `-w` flag).
